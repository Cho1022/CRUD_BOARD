# 프로젝트 수행 규범

1. 항상 한국어로 답한다.
2. 기존 루트 `backend`, `frontend`, `docker-compose.yml`은 덮어쓰거나 삭제하지 않는다.
3. 새 코드는 `게시판/front`, `게시판/backend` 하위에만 만든다.
4. 위험 작업은 즉시 중단하고 사용자에게 확인한다.
5. 위험 작업에는 전역 설치, 전역 설정 변경, Docker 볼륨 삭제, 기존 DB 삭제가 포함된다.
6. 외부 유료 API 호출, OpenAI API 호출, API Key 사용은 금지한다.
7. 클라우드 배포, 외부 저장소 연결, Git push는 사용자 확인 없이 하지 않는다.
8. 백엔드는 TDD를 우선한다.
9. 테스트는 Red -> Green -> Refactor 순서로 진행한다.
10. 프론트는 유효성 검사와 핵심 컴포넌트 위주로 테스트한다.
11. 메서드는 12줄 이하를 목표로 한다.
12. 조건문/반복문 depth는 3 이하를 목표로 한다.
13. 과한 분리로 가독성이 나빠지면 코드 규칙보다 명확성을 우선한다.
14. Lombok을 사용해 보일러플레이트를 줄인다.
15. 지역 변수는 가독성이 좋아질 때 Lombok `val`을 사용한다.
16. Controller에는 비즈니스 로직을 두지 않는다.
17. DTO는 각 도메인의 `dto` 폴더 바로 아래 둔다.
18. `request`, `response` 하위 폴더는 만들지 않는다.
19. 검증은 프론트와 백엔드 모두에 둔다.
20. 응답 구조는 `ApiResponse`, 페이지 응답은 `PageResponse`로 통일한다.
21. 게시글과 댓글은 소프트 삭제한다.
22. 태그 추천은 외부 API 없이 로컬 키워드 기반으로 구현한다.
23. 이미지 저장은 `FileStorage` 인터페이스를 통해 분리한다.
24. 1차 구현은 `LocalFileStorage`만 사용한다.
25. RAG/GraphRAG/MCP는 1차 구현에 넣지 않고 확장 가능성만 유지한다.
